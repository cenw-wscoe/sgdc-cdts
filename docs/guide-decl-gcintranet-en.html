GCIntranet / version / default

Section <head> references

  • Did you add the two references to the CDTS files (cdts[-esdc]-styles.css / wet-[language].js)?
  • Did you implement a version mechanism (web.config, XML file, database configuration, etc...) to control the URL, theme and version of the CDTS?
  • Did you add the <noscript> section?
  • Did you insert the correct backup static file into the <noscript> you created?
  • Did you add the data-cdts-setup attribute to specify the CDTS configuration object OR called the wet.builder.setup function?
  • For details, refer to the implementation guide.

Section top references

  • Did you add the new <div id="cdts-def-top"> ?
  • Did you insert the correct backup static file into the <div id="cdts-def-top"> you created?
  • Did you add a top object to the CDTS configuration object for any customization, including subTheme if any?
  • For details, refer to the implementation guide.

Section pre-footer references

  • Did you add the new <div id="cdts-def-prefooter">?
  • Did you insert the correct backup static file into the <div id="cdts-def-prefooter"> you created?
  • Did you add a preFooter object to the CDTS configuration object for any customization?
  • For details, refer to the implementation guide.

Section footer references

  • Did you add the new <div id="cdts-def-footer">?
  • Did you insert the correct backup static file into the <div id="cdts-def-footer"> you created?
  • Did you add a footer object to the CDTS configuration object for any customization?
  • For details, refer to the implementation guide.

Important note for application version

The following two sections are for the application versions. Keep in mind this is different than removing or adding certain elements that are allowed as per the C&IA specifications document. Any element that can be implemented using the C&IA specifications document does not require a custom header/footer and is available in the default version of the CDTS (see above). Currently it is advisable to not implement a custom header/footer unless you have permission to do so from your department, TBS or Principal Publisher.
