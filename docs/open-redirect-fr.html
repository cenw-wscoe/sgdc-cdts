Vulnérabilité de redirection ouverte

Note

Si vous utilisez actuellement la fonctionnalité exitscript avec une page de redirection (à l'aide du paramètre exitURL), il existe une vulnérabilité de sécurité qui doit être corrigée.

Si vous extrayez l'URL du paramètre targetURL dans la chaîne de requête du URL, vous DEVEZ valider l'URL avant de rediriger l'utilisateur.

La validation doit garantir que l'URL est vérifié par rapport à une liste d'URL acceptés.

Pour un exemple C#, visitez le wiki des modèles .NET.

REMARQUE : ceci s'applique à la version SGDC v4.1.0 et antérieure. La nouvelle version de SGDC n'utilisera pas le paramètre targetURL.
