Configurations de modèle Web SGDC

Slogan

Note

Le modèle Web offre plusieurs façons de configurer votre site et son comportement pour répondre à vos besoins. Vous trouverez ci-dessous les éléments qui peuvent être configurés soit statiquement via les entrées de configuration (web.config pour .NET ou goc.webtemplate.global.config.cdn.properties pour Java) ou par programmation. Les exemples de code pour chacun d'entre eux se trouvent dans les fichiers zip des modèles Web et également sur NuGet pour l'équipe EDSC .Net.

Configurations de modèles Web

Environnement

Ce paramètre détermine l'environnement CDTS que vous utiliserez pour utiliser les ressources Web.

  • Définissez dans la configuration :
    • GoC.WebTemplate.Environment :
      • Spécifie le SGDC à utiliser
      • 6 valeurs sont disponibles :
        • AKAMAI :
          • Thème GCWeb :
            • GCWeb est le seul thème supporté dans cet environnement
            • Doit être utilisé par toutes les applications publiques.
            • Le même environnement doit être utilisé pour votre développement, vos tests et votre mise en production.
        • PROD_SSL :
          • GCIntranet est le seul thème supporté dans cet environnement
            • Doit être utilisé par toutes les applications GoC Intranet.
        • PROD_UNSECURE :
          • GCIntranet est le seul thème supporté dans cet environnement
            • Doit être utilisé par toutes les applications GoC Intranet.
        • ESDC_PROD :
          • GCIntranet est le seul thème supporté dans cet environnement
            • Doit être utilisé par toutes les applications internes de EDSC.
            • Doit être utilisé seulement pour votre mise en production.
            • Utilisez « ESDCNonProd » pour votre développement et vos tests.
        • Environnements SADE :
          • ESDC_GCWEB_SADE :
            • Bientôt ce SGDC ne supportera plus ce thème. Actuellement, il soutient uniquement la migration vers le SGDC d'Akamai.
            • Les consommateurs de GCWeb doivent utiliser l'environnement « Akamai ».
          • ESDC_GCIntranet_SADE :
            • Doit être utilisé par toutes les applications internes de EDSC.
            • Ce SGDC n'est pas accessible aux services externes.
            • À utiliser uniquement pour votre mise en œuvre de développement et de test.
            • Utilisez « ESDCProd » pour votre mise en production.
        • Environnements d'AQ d'EDSC :
          • ESDC_GCIntranet_QA :
            • Bientôt ce SGDC ne supportera plus ce thème. Actuellement, il soutient uniquement la migration vers le SGDC d'Akamai.
            • Les consommateurs de GCWeb doivent utiliser l'environnement « Akamai ».
          • ESDC_GCWEB_QA :
            • Doit être utilisé par les applications internes de EDSC.
            • Ce SGDC n'est pas accessible aux services externes.
            • Peut être utilisé pour vos tests AQ.

Charger JQuery depuis Google

Ce paramètre détermine si les fichiers JQuery doivent être récupérés depuis Google ou depuis le SGDC spécifié dans le paramètre « Environment » de modèles Web (Akamai, ESDCProd etc...).

  • Définissez dans la configuration :
    • GoC.WebTemplate.loadJQueryFromGoogle
    • Valeurs :
      • true : récupérera les fichiers de google.
      • false : récupérera les fichiers du SGDC spécifié dans le paramètre « Environnement ».
  • Peut également être défini par programmation via la propriété « LoadJQueryFromGoogle » du modèle Web.

Sous-thème

Ce paramètre déterminera le sous-thème des actifs Web à utiliser à partir du SGDC. Exemple : Le thème GCIntranet supporte le thème « GC » et le thème « esdc ».

  • Définissez dans la configuration :
    • GoC.WebTemplate.SubTheme
    • Valeurs :
      • vide (valeur vide). Est la valeur par défaut.
      • edsc
  • Peut également être défini par programmation via la propriété « WebTemplateSubTheme » du modèle Web.

Utiliser HTTPS

Cela ne peut être défini que si l'option « IsSSLModifiable » est définie dans le fichier CDTSEnvironments.json , sinon elle doit être nulle ou non définie dans la configuration

Ce paramètre détermine si la communication entre le navigateur et le SGDC doit être cryptée.

  • Définissez dans la configuration :
    • GoC.WebTemplate.useHTTPS
    • Valeurs :
      • true : utilisera https
      • false : utilisera https
      • vide : https non disponible
  • Peut également être défini par programmation via la propriété « UseHTTPS » du modèle Web.

Version

Ce paramètre déterminera la version des ressources Web à utiliser à partir du SGDC. Les modifications apportées aux actifs Web hébergés sur le SGDC généreront une nouvelle version de ceux-ci. La gestion des versions permet au SGDC de déployer les modifications dès que possible et offre aux applications la flexibilité de passer à la dernière version dès que possible. Le modèle peut également prendre en charge la version « exécutable » des ressources Web SGDC, cela signifierait que vous n'utilisez pas les fichiers versionnés, mais que vous utilisez les derniers fichiers au fur et à mesure qu'ils sont mis à jour.

  • Définissez dans la configuration :
    • GoC.WebTemplate.Version
    • ex: v4_0_17
    • laissez vide pour utiliser les derniers fichiers SGDC et non les fichiers versionnés. Vous devrez également fournir les fichiers de secours statiques « run ». [voir la section des « configurations » dans le wiki DotNetTemplates pour plus de détails.].
  • Peut également être défini par programmation via la propriété « WebTemplateVersion » du modèle Web.

Configurations d'en-tête

Titre de l'application

Lorsque vous utilisez le thème « GCIntranet » dans votre application, vous pouvez éventuellement définir le texte du titre de l'application et son URL. Le titre de l'application s'affiche en haut de la page, au-dessus du menu supérieur.

  • Défini par programmation via la propriété « ApplicationTitle_Text » du modèle Web.
  • L'URL du titre de l'application est facultative. Si laissé vide, le sous-thème déterminera l'URL à utiliser. Vous pouvez remplacer la valeur d'URL par défaut du sous-thème en définissant par programmation la propriété « ApplicationTitle_URL » du modèle Web.

« Breadcrumbs »

Les fils d'Ariane sont une liste de liens affichés près du haut du contenu pour aider l'utilisateur à déterminer où il se trouve sur le site et pour l'aider à naviguer. En mode transactionnel, le fil d'Ariane doit être remplacé par un lien pour quitter la transaction, qui dirige l'utilisateur vers un point sûr où il peut abandonner la transaction en cours et la redémarrer s'il le souhaite.

  • Définissez par programme en renseignant la collection « Breadcrumbs » du modèle Web. Le « Breadcrumbs » a 3 propriétés :
    • « href » : l'url du lien. S'il est vide, l'élément sera affiché sous forme de texte et non sous forme de lien hypertexte.
    • « title » : le texte affiché à l'écran.
    • « acronym » : si votre titre a un acronyme, vous pouvez utiliser cette propriété pour fournir le texte complet du titre. Il sera affiché lorsque l'utilisateur survolera le lien.

Titre de l'en-tête

Ce paramètre détermine le titre de votre site en remplissant la balise de <title>.

  • Défini par programmation via la propriété « HeaderTitle » du modèle Web.

« MetaTags »

Le modèle Web fournit une collection de « chaînes » pour inclure des éléments HTML, y compris des balises Meta, à la section d'en-tête de la page. Cette collection peut être utilisée par votre application pour inclure toute balises Meta requise par votre site.

  • Définissez par programmation en renseignant la collection « HTMLHeaderElements » du modèle Web.
  • La collection est de type « String », et la balise html complète que votre MetaTag doit inclure doit être fournie. ex : "<meta name="chanteur" content="Elvis">"

Pré contenu

Celui-ci sera utilisé par Portfolio Web pour insérer du contenu dans l'espace de pré-contenu de votre page. Par défaut, ceci est TOUJOURS affiché sur toutes les pages. Vous aurez besoin de l'autorisation de Portfolio Web ou de l'éditeur principal pour ne pas inclure ce contenu dans votre ressource Web.

  • Définissez dans la configuration :
    • GoC.WebTemplate.ShowPreContent:
    • Valeurs :
      • true (affiché)
      • false (non affiché)
  • Peut également être défini par programmation via la propriété « ShowPreContent » du modèle Web.

Recherche

La recherche est le contrôle de recherche GC qui s'affiche en haut de la page. Par défaut, la recherche est TOUJOURS affichée sur toutes les pages (sauf pour le mode transactionnel). Vous aurez besoin de l'autorisation de Portfolio Web ou de l'éditeur principal pour ne pas inclure ce contenu dans votre candidature.

  • Définissez dans la configuration :
    • GoC.WebTemplate.ShowFeatures:
    • Valeurs :
      • true (affiché)
      • false (non affiché)

Configurations du contenu principale

Menu de gauche

Votre application peut afficher un menu de gauche pour faciliter la navigation en sélectionnant la page maître appropriée pour vos pages de contenu. Les éléments du menu sont contrôlés par programmation et 3 niveaux d'éléments de menu sont pris en charge.

  • Définissez par programme en remplissant la collection « LeftMenuItems » de « MenuSection » du modèle Web. La « MenuSection » a 3 propriétés :
    • « Link » : le lien de l'en-tête de la section de menu.
    • « Name » : le nom de l'en-tête de la section de menu.
    • « OpenInNewWindow » : par défaut est false, mais s'il est défini sur true, le lien s'ouvrira dans une nouvelle fenêtre.
    • « List of Link » : représente les liens pour la section de menu.
  • Lorsque « OpenInNewWindow » est défini sur true, une balise span sera incluse pour l'accessibilité et pour informer l'utilisateur que le lien ouvrira une nouvelle fenêtre.

Configurations de pied de page

À propos des liens

Les liens À propos de nous dans le pied de page sont prédéterminés par le thème utilisé pour votre site. Certains thèmes permettent de les remplacer par votre propre liste. Si vous décidez de fournir votre liste, les liens prédéterminés du thème ne seront pas affichés.

« Remarque » : ne s'applique pas à tous les thèmes.

  • Définissez par programme en remplissant la collection « AboutLinks » du modèle Web. Cette collection est de type « Lien », qui possède 2 propriétés :
    • « href » : l'url du lien
    • « texte » : le texte affiché à l'écran

Liens de contact

Les liens « Contactez-nous » dans le pied de page sont prédéterminés par le thème. Certains thèmes permettent de les remplacer par votre propre liste. Si vous décidez de fournir votre liste, les liens prédéterminés du thème ne seront pas affichés.

Remarque : ne s'applique pas à tous les thèmes.

  • Définissez par programmation en renseignant la collection "ContactLinks" du modèle Web. Cette collection est de type "Lien", qui possède 2 propriétés :
    • « href » : l'url du lien
    • « texte » : le texte affiché à l'écran

Date de modification

La date de modification s'affiche en bas de la page. Cet élément partage le même emplacement que « VersionIdentifier » et un seul des deux peut être affiché à la fois. Si un « DateModified » est fourni, la date prévaudra sur le « VersionIdentifier ».

Défini par programmation via la propriété « DateModified » du modèle Web.

Fonctionnalités

Les fonctionnalités sont les activités et les initiatives du GC qui sont affichées au bas du contenu. Par défaut, les fonctionnalités sont TOUJOURS affichées sur toutes les pages. Vous aurez besoin de l'autorisation de Portfolio Web ou de l'éditeur principal pour ne pas inclure ce contenu dans votre ressource Web.

  • Définissez dans la configuration :
    • GoC.WebTemplate.ShowFeatures:
    • Valeurs :
      • true (affiché)
      • false (non affiché)

Post contenu

Cela sera utilisé par Portfolio Web pour insérer du contenu dans l'espace de contenu de publication de votre page. Par défaut, ceci est TOUJOURS affiché sur toutes les pages. Vous aurez besoin de l'autorisation de Portfolio Web ou de l'éditeur principal pour ne pas inclure ce contenu dans votre ressource Web.

  • Définissez dans la configuration :
    • GoC.WebTemplate.ShowPostContent:
    • Valeurs :
      • true (affiché)
      • false (non affiché)
  • Peut également être défini par programmation via la propriété « ShowPostContent » du modèle Web.

Identificateur d'écran

L'identifiant d'écran est affiché près du bas de la page. Cet élément sert d'identifiant unique pour votre page et peut être utilisé pour faciliter la communication entre les utilisateurs et l'équipe de support/service d'assistance afin d'identifier l'emplacement exact d'un utilisateur dans votre application.

  • Défini par programmation via la propriété « ScreenIdentifier » du modèle Web.

Identifiant de version

L'identificateur de version est affiché près du bas de la page. Cet élément partage le même emplacement que « DateModified » et un seul des deux peut être affiché à la fois. Si un « DateModified » est fourni, la date prévaudra sur le « VersionIdentifier ».

  • Défini par programmation via la propriété « VersionIdentifier » du modèle Web.

Configurations spécifiques au modèle transactionnel

Lien « Confidentialité »

Le lien de confidentialité s'affiche dans la présentation transactionnelle au bas de la page. Le thème définit l'URL par défaut, mais l'URL peut être définie par programme.

  • Défini par programmation via la propriété « PrivacyLink_URL » du modèle Web.

Lien « Conditions générales »

Le lien Termes et conditions s'affiche dans la présentation transactionnelle au bas de la page. Le thème définit l'URL par défaut, mais l'URL peut être définie par programme.

  • Défini par programmation via la propriété « TermsConditionsLink_URL » du modèle Web.x

Configurations spécifiques au thème GCWeb

« Commentaires » - Lien « Signaler un problème »

Le lien Signaler un problème (anciennement le lien « Commentaires ») est un moyen pour les utilisateurs de votre site de fournir des commentaires sur leur expérience de navigation sur le site ou de signaler un problème. L'affichage du lien est facultatif mais il ne peut pas être contextualisé et Portfolio Web ou l'éditeur principal doivent être contactés pour vous aider à déterminer s'il doit être affiché sur votre site. L'URL du lien est prédéfinie par le thème mais peut être modifiée dans le web.config ou par programmation.

  • Définissez dans la configuration :
    • GoC.WebTemplate.ShowFeedbackLink:
    • Valeurs :
      • true (affiché)
      • false (non affiché)
  • Peut également être défini par programmation via la propriété « ShowFeedbackLink » du modèle Web.
  • Si vous souhaitez rediriger l'utilisateur vers votre propre page au lieu de la page par défaut de canada.ca, définissez la configuration « FeedbackLinkURL » dans le fichier web.config sur l'url de votre page, ou définissez la propriété « FeedbackLinkURL » par programmation.

Lien « Partager cette page » :

Le lien « Partager cette page » est un moyen pour les utilisateurs de partager l'URL du site via les médias sociaux. L'affichage du lien est facultatif et Portfolio Web ou l'éditeur principal doit être contacté pour vous aider à déterminer s'il doit être affiché sur votre site et quel site de médias sociaux doit être présent.

  • Définissez dans la configuration :
    • GoC.WebTemplate.ShowSharePageLink:
    • Valeurs :
      • true (affiché)
      • false (non affiché)
  • Peut également être défini par programmation via la propriété « ShowSharePageLink » du modèle Web.

Si vous décidez d'afficher le lien, vous devez fournir la liste des sites de médias sociaux à afficher à l'utilisateur en renseignant par programmation la collection « SharePageMediaSites » du modèle Web. Si la collection « SharePageMediaSites » est laissée vide, le lien ne sera pas affiché. La collection est de type « SocialMediaSites » qui est une énumération du modèle Web.
