GCIntranet / version / normal

Références de la section <head>

  • Avez-vous ajouté les deux références aux fichiers du SGDC (cdts[-esdc]-styles.css / wet-[language].js)?
  • Avez-vous implémenté un mécanisme de version (web.config, fichier XML, configuration en base de données, etc...) pour contrôler le URL, thème et version du SGDC?
  • Avez-vous ajouté la section <noscript>?
  • Avez-vous inséré le contenu statique de repli dans cette section <noscript>?
  • Avez-vous ajouté l'attribut data-cdts-setup pour spécifier l'objet de configuration SGDC OU appelé la fonction wet.builder.setup?
  • Pour plus de détails, se référer au guide de mise en œuvre.

Références de la section top

  • Avez-vous ajouté le nouvel élément <div id="cdts-def-top"> ?
  • Avez-vous inséré le fichier statique de repli dans l'élément <div id="cdts-def-top">?
  • Avez-vous ajouté un objet top dans l'objet de configuration SGDC pour personnalisation, incluant le sous-thème selon le cas?
  • Pour plus de détails, se référer au guide de mise en œuvre.

Note importante pour gabarit pour applications

Les deux sections suivantes sont pour le gabarit pour applications. N'oubliez pas que l'utilisation du gabarit d'application est différente de la simple suppression ou de l'ajout de certains éléments permis par les Spécifications du contenu et de l’architecture de l’information pour Canada.ca.. Tout élément qui peut-être implémenté selon la spécification Canada.ca ne nécessite pas un entête/pied de page personnalisé et est disponible dans le gabarit normal de SGDC (voir ci-haut). Avant d'aller de l'avant, assurez-vous d'avoir la permission de votre département, du SCT ou de l’éditeur principal.
