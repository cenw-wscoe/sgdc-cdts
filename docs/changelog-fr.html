Annonces (obsolètes)

  • Si vous avez précédemment implémenté les gabarits d'application, vous ne pourrez PAS mettre à niveau sans mettre à jour votre code. Une mise à jour obligatoire des fichiers statiques de sauvegarde sera également requise. Veuillez consulter GCpédia pour le code mis à jour et les fichiers statiques. * RL
  • Dans les deux thèmes, la version /rn/ est maintenant une copie de la version /app/. Cela réduit le temps de développement et assure une plus grande cohérence entre les deux versions. * RL
  • Tous les fichiers /rn/ statiques (GCintranet et GCweb) doivent être mis à jour avant BEOW v4.1.0. * RL
  • Toutes les implémentations de GCWeb sur ssl-templates.services.gc.ca cesseront de fonctionner sous peu, assurez-vous que vous utilisez les URL d'Akamai. * RL
  • Les serveurs Akamai pour CDTS proposent désormais des fichiers HTML statiques. Assurez-vous de récupérer les fichiers auprès d'Akamai et non de ssl-templates.services.gc.ca. * RL
  • Dans v4.0.24 - v4.0.25, le code HTML du menu supérieur dans les gabarits d'application a changé (GCIntranet uniquement), car nous ne connaissons personne à l'exception de ESDC EI ORT web asset utilisant les gabarits d'application sur le thème GCIntranet que nous avons mis à jour automatiquement. Si vous avez été concerné, n'oubliez pas que vous devez informer l'équipe EDSC WS CoE que vous l'utilisez afin que nous puissions vous avertir à l'avance de toute modification des gabarits d'application. * RL
